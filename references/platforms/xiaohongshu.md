# 小红书账号 / 内容诊断子手册

> 主 skill：`social-account-doctor`。本手册被 §3 平台专属诊断章节调用。

## 0.5 算法规则速查 — T0/T1（可执行版）

> ⭐ **算法核心信号 = 互动率（CES 综合，重收藏 + 评论）**。差距 ≥ 1 档时**优先于其他维度修** — 小红书算法对收藏 / 评论 / 转发权重极高（CES = 点1+藏1+评4+转4+关8）。
>
> T0 = 不做就死（直接卡漏斗）；T1 = 不做次推（ROI 最高的优化项）。

| 级 | 规则 | 怎么做（动作） | 不做的代价 |
|---|---|---|---|
| ⭐ **T0** | 内容必须有"留着用"价值（驱动收藏） | 改成清单 / 教程 / 避坑 / 资源至少一种，结尾写"建议收藏" | 收藏/点赞 < 0.3 = 假爆款，CES 直接腰斩 |
| ⭐ **T0** | 结尾必埋开放性互动钩子（驱动评论） | 「你们怎么解决？」/「评论扣 1 我发资料」 | 评论率 < 0.5%，CES 进不了下一档 |
| **T0** | 标题前 10 字必含核心搜索词 | 「治愈系穿搭｜新手 3 步速成」←核心词放最前；不要「今天分享一个超治愈的搭配」 | 失去近 70% 月活的搜索入口（COO 公开口径） |
| **T0** | 封面 = 3:4 竖版 + 大字占图 ≥ 1/4 + 高对比 | 暖底冷字 / 黑黄 / 红白；字号至少占封面 1/4，重点字加描边 | CTR < 5%，进不了二级流量池 |
| **T1** | 首段 30 字内出钩子 | 首句直接抛痛点 / 数字 / 反认知，砍"分享给大家" | 阅读完成率 < 30%，正文白写 |
| **T1** | 标签按公式：1 大词 + 2 垂直词 + 1 人群词 + 1 时效词 | 共 5 个，避免堆砌同义词 | 搜索分发崩 |
| **T1** | 用"姐妹 / 兄弟 / 朋友"语气，不用蓝 V 腔 | 第一人称口语；不要"亲爱的用户们" | 算法对官方语气敏感 |

## 1. 阈值表（行业经验值 + 半官方口径）

| 指标 | 计算 | 偏低 | 合格 | 优秀 | 卡住意味着 |
|---|---|---|---|---|---|
| 曝光率 | 24h 曝光 / 粉丝数 | < 0.5 | 0.5-2.0 | > 2.0 | 限流 / 标签错乱 / 账号权重低 |
| **封面 CTR** | **点击 / 曝光** | **< 5%** | **5-10%** | **> 12%** | **封面不及格（首图定生死）** |
| 阅读完成率 | 完整读完 / 点击 | < 30% | 30-50% | > 50% | 首图后无信息密度 / 排版差 / 正文太水 |
| 互动率 | (赞+藏+评+转) / 曝光 | < 3% | 3-5% | > 5%（迅课经验：≥5% 进二级流量池） | 选题没共鸣 / 没埋互动钩子 |
| **收藏/点赞比** | **收藏 / 点赞** | **< 0.3**（假爆款） | **0.5-1**（健康） | **> 1**（爆款实用类） | **收藏低=没"留着用"价值，长尾完蛋** |
| 搜索流量占比 | 搜索曝光 / 总曝光 | < 30% | 30-50% | > 50% | 关键词埋点不足 / 标签错（半官方："近 70% 月活有搜索行为"） |
| 关注转化率 | 新增关注 / 曝光 | < 0.05% | 0.1-0.3% | > 0.3% | 人设模糊 / 无复购理由 |

### 1.1 CES（社区参与度分数，业界推断公式 — 非小红书官方公布）

```
CES = 点赞×1 + 收藏×1 + 转发×4 + 评论×4 + 关注×8
```

**社区共识来源**：新红数据 / 飞书文档 / 凡象品牌等多源一致。但小红书官方从未确认此系数，**用作健康度估算而非绝对评判**。

**经验铁律**：
- **收藏权重 ≈ 点赞**（不是 0.5），因为收藏代表"长期价值"
- **评论 + 转发 + 关注是高权重信号**，一条爆款笔记如果只有点赞没有这三项 = 假爆款风险高
- 健康笔记结构（行业经验）：**收藏/点赞 ≥ 0.5**（实用类）；**评论/点赞 ≥ 0.05**（共鸣类）

### 1.2 封面 CTR 阈值（行业经验值）

| 笔记类型 | 偏低 | 合格 | 优秀 |
|---|---|---|---|
| 图文笔记（首图竖版 3:4） | < 5% | 5-10% | > 12% |
| 视频笔记（动态首帧） | < 4% | 4-8% | > 10% |
| 商品笔记（带价格标） | < 3% | 3-6% | > 8% |

**经验铁律**：小红书封面是**静态大字报**，不是抖音的动态首帧。**3:4 竖版 + 大字 + 高对比** 是基本功。

## 2. 主逻辑 1：CES + 封面 CTR 诊断（小红书头号杀手）

### 2.1 怎么定位是不是卡在这一层

```
1. 漏斗看到 CTR < 5% 或 CES 进不了下一档 → 直接进本节
2. 调 tikhub xiaohongshu xiaohongshu_web_v2_fetch_feed_notes_v2(note_id)
   → 拿封面 url + 标题 + 互动数据（赞/藏/评/转）+ 阅读完成率
3. 下载封面到 /tmp/account_diagnostic/{note_id}/cover.jpg
4. 调 scripts/analyze_image.py（拆封面 5 变量 + 模板归类）
   → 拿 big_text / big_text_ratio / human_presence / color_contrast / info_density
   → 拿 template_classification (A/B/C/D/E) + hook_detection
5. 对照 §2.2 五种封面失败模式定位根因
6. 同时算 CES 各分项贡献，看是哪一项被拖累（赞高但藏低 = 留不住 / 评低 = 没共鸣）
```

### 2.2 封面 CTR 崩盘的五种典型根因

| 根因 | 现象 | 修法 |
|---|---|---|
| **大字不够大** | big_text_ratio < 0.15（字占封面 < 15%） | 字号至少占封面 1/4，重点字加描边 |
| **颜色对比弱** | color_contrast = "同色系（弱）" | 暖底冷字 / 黑黄 / 红白，避免同色系 |
| **无人群锚定** | hook_detection.audience_lock 为空 | 封面加"职场人/学生党/新手妈妈"等明确人群词 |
| **信息密度低** | info_density = "低" | 加"3 个/省 500/避坑" 等利益数字 |
| **模板错位** | matched 是 D（实物展示）但赛道流行 A（大字报） | 看赛道 top 10 封面模板分布，对齐主流模板 |

### 2.3 CES 分项诊断（找出短板项 — 业界推断阈值，非官方）

| 信号 | 含义 | 修法 |
|---|---|---|
| **收藏 < 点赞 × 0.3** | 内容没有"留着用"的价值 | 改为清单/教程/资源贴，加"建议收藏" |
| **评论 < 点赞 × 0.02** | 没共鸣 / 没钩子 | 结尾埋开放性问题（"你们怎么解决？"） |
| **转发 < 点赞 × 0.05** | 不值得分享出去 | 改成"避坑/拯救/我帮你做完了"句式 |
| **关注 < 互动总数 × 0.5%** | 内容好但人设不立 | 主页/简介/前 5 条对齐统一人设 |

## 3. 主逻辑 2：搜索分发诊断（小红书命脉，COO 公开"近 70% 月活有搜索行为"）

### 3.1 怎么定位是不是卡在这一层

```
1. 漏斗看到 搜索流量占比 < 30% → 直接进本节
2. 调 tikhub xiaohongshu xiaohongshu_web_v2_fetch_feed_notes_v2(note_id)
   → 拿标题 + 正文 + 标签 + 话题
3. 提取所有出现的关键词（分词 / 命名实体）
4. 算"目标关键词"在标题/正文/标签三处的出现次数
5. 调 tikhub xiaohongshu xiaohongshu_app_search_notes(keyword=目标词, sort_type=general)
   → 看自己的笔记排在第几位（搜不到 = 关键词埋点失败）
6. 对照 §3.2 搜索失败的四种典型根因
```

### 3.2 搜索分发崩盘的四种典型根因

| 根因 | 现象 | 修法 |
|---|---|---|
| **关键词埋点过少** | 标题、正文、标签合计目标词出现 < 3 次 | 标题必出现 1 次，正文 ≥ 2 次（首段+中段），标签 ≥ 1 次 |
| **只用大词不用长尾词** | 全是"减肥"，没有"产后减肥/办公室减肥" | 大词 1 个 + 长尾词 2-3 个组合（覆盖搜索意图） |
| **关键词堆砌触发反作弊** | 同关键词高频出现 / 标签塞满同义词 | **注意**：公开源仅有"堆砌会限流"的定性说法，无具体次数阈值。建议自然分布、每段 1 次足够 |
| **标签选择错误** | 标签全是赛道大词（#减肥 #健身），无垂直长尾 | 标签 = 1 个赛道大词 + 2 个垂直词 + 1 个人群词 + 1 个时效词（白杨 SEO 经验：5-10 个为宜） |

### 3.3 关键词布局公式（小红书 SEO 黄金法则）

```
标题：1 个核心词（前 10 字内）+ 1 个修饰词（数字/情绪/人群）
正文首段：核心词 + 同义词出现 1 次
正文中段：核心词出现 1 次（在小标题或加粗位置）
正文尾段：相关长尾词 1-2 次
标签：1 大词 + 2 垂直词 + 1 人群词 + 1 时效词（共 3-5 个）

总计：核心词 ≥ 3 次（不超 5 次），长尾词 2-3 个
```

## 4. tikhub-xiaohongshu 工具映射表

> 首选 / Fallback。首选挂时按列依次重试。
>
> **⚠️ 硬规则（实测 2026-04-21，与 SKILL.md §9 接口稳定性表对齐）**：
> - **小红书所有 V2 接口（不论 app v2 还是 web v2）当前全挂**（RetryError[HTTPStatusError]）—— 直接用 V1，**不要再去试 V2,也不要"V2 失败切 web V2"**（旧建议已过时,实测两个都挂）
> - 关键词搜笔记 = `xiaohongshu_app_search_notes`（App V1，唯一稳定）
> - 关键词搜用户 = `xiaohongshu_web_search_users`（Web V1，唯一稳定 — 注意 App V1 `search_users` 也挂）
> - 任一工具返回 `RetryError[HTTPStatusError]` = 直接按 SKILL.md §9 表换可用版本，**禁止在同一接口刷重试**（tenacity 已重试 3 次了，再刷只会浪费时间 + 计费）
> - 返回 `code != 200` 但有 `message` → 看 `message_zh`，是参数错 / 限流 / 余额不足，按提示改参数；不要当成"接口挂了"换工具
> - 返回 `data.items` 为空 → 关键词无结果或被风控，换近义词/降低 sort_type

### 4.1 推荐调用顺序（输入 = 账号链接时）

```
1. dispatch_account.py <url> → 拿到 user_id (+ xsec_token)
2. xiaohongshu_web_v2_fetch_user_info(user_id) → 账号基础信息
3. xiaohongshu_app_v2_get_user_posted_notes(user_id, cursor="") → 最近 20 条 note_id 列表
4. 计算每条互动数据（avg_like, avg_collect, avg_comment）→ 算 CES 均值 + 爆款率
5. 取 top 3 + bottom 3 → 对每条调 xiaohongshu_web_v2_fetch_feed_notes_v2 → 拿完整数据
6. 对 top 3 + bottom 3 → scripts/analyze_image.py 拆封面 5 变量
7. 对 top 3 → xiaohongshu_web_v2_fetch_note_comments 拿前 20 条评论（提炼痛点）
8. 关键词搜索 → 找对标候选（5k-50k 粉，活跃，同赛道）
```

## 5. 小红书版六维评分细则

| 维度 | 小红书特化指引 |
|---|---|
| **账号定位** | 主页 9 宫格风格统一性 + 简介人群锁定 + 是否有"专攻 XX"的标签词 |
| **选题角度** | **必须有搜索量背书**：调 `xiaohongshu_app_search_notes` 验证目标词是否有 ≥ 100 笔记，避免无人搜的伪需求 |
| **封面公式** | 套 SKILL.md §5.1 五种封面（A 大字报 / B 对比 / C 真人出镜 / D 实物展示 / E 表格截图）。**3:4 竖版 + 大字 ≥ 1/4 + 高对比** 是基本功 |
| **标题钩子** | 套 SKILL.md §5.2 10 公式 + **强制带核心关键词**（前 10 字内）+ 数字/emoji/【】 |
| **正文骨架** | **首段 30 字内出钩子**（提问/承诺/反差）→ 3-5 个分点（emoji 分隔）→ 结尾互动引导 + 标签 |
| **发布节奏** | 小红书算法**长尾性强**，频率 2-3 条/周即可。**发布时段**：工作日 11-13 / 19-22；周末 9-11 / 15-17 / 20-23 |

## 7. 关键铁律（小红书专属）

1. **收藏 ≈ 点赞，且收藏低 = 长尾完蛋**。点赞高但收藏低 = 假爆款风险高。
2. **搜索意图占比高**（COO 柯南 2025 公开"近 70% 月活有搜索行为"）。关键词埋点比抖音重要 10 倍，但**堆砌会限流**（具体阈值无官方数字，自然分布即可）。
3. **封面是静态大字报，不是动态首帧**。3:4 竖版 + 大字 + 高对比是基本功。
4. **CES 是业界推断的健康度指标**（非官方），用作"对标 vs 我方"差距分析，别当成绝对死线。
5. **首段 30 字内必出钩子**。小红书读者比抖音用户更耐心，但首屏不抓人照样划走。
6. **长尾效应 1-2 年**。一篇好笔记可以持续带流量，但前提是关键词布局对了。
7. **不要用蓝 V 风格**。小红书算法对官方语气敏感，越像"姐妹分享"越推。
